Dans bot/src/scheduler.js, dans updateReminderMessages(), avant de créer les nouveaux messages, vérifier si les IDs existants dans le cache/Supabase pointent vers des messages Discord valides :
Si un message existe déjà (ID trouvé) → l'éditer comme avant

Si un message n'existe pas (ID absent ou message Discord introuvable/supprimé) → créer un nouveau
Le problème actuel : au redémarrage, les anciens messages restent et de nouveaux sont créés en plus.
Fix : au démarrage du scheduler dans startScheduler(), faire un nettoyage du salon REMINDER_CHANNEL_ID :

Fetch les IDs reminder_dr1_msg, reminder_dr2_msg, reminder_raid_msg depuis Supabase
Pour chaque ID, vérifier si le message Discord existe encore
Supprimer physiquement tous les messages du salon qui ne correspondent PAS à ces IDs (les orphelins)
Comme ça au redémarrage, les vieux messages sans ID Supabase sont nettoyés automatiquement
